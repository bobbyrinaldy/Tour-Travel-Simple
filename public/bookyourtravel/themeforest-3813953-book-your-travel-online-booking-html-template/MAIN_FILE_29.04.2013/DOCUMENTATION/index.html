Instalation Top

Book Your Travel is a HTML template so there's no really "installation" required just unzip the main file you've downloaded from themeforest and after unziping you have your all template files ready to be used.

File Structure Top

  • css folder: contains css stylesheet files. In these files it is defined how to display HTML elements.
  • fonts folder: contains a @font-face font kit that are loaded via css file. Explanation on changing fonts is in the "fonts section".
  • images folder: contains all necessary images for website to display the same than in .png files. The file is separated into several files:
    • bgr folder: contains background related images
    • ico folder: contains icon related images
    • txt folder: contains text related images
    • uploads folder: contains content related images
    • slider folder: contains slider related images
  • js folder: contains javascript files. These files provide enhanced user experience and dynamic effects.

HTML Structure Top

Html head section

The <head> element is a container for all the head elements. The head section contains elements that define the page description, keywords, and other SEO meta information. It also includes links to the stylesheets that define the colors and styling of the page. See changing color styles for a instructions on how to change the color of the template.

To edit the title, description or keywords of your page, open index.html or any other page for that matter and edit the text within the "content" element:

				<head>
					
					Book Your Travel
					
					
					
					
					
					
					
					
					
					
					
					
				</head>

					$(".form:first").show(); //on home page first form is visible by default

Fonts Top

@font-face is a css rule which allows you to download a particular font from your server to render a webpage if the user hasn't got that font installed.

The Basic Syntax

@font-face only requires a few lines of CSS and is therefore very easy to use. To get started, stop by one of the free font sites and download any font you like.

Once you’ve downloaded the font, place the font file in the fonts directory.

Cross-Browser Compatibility

All the major browsers have decided to go their own way with font formats that they choose to support:

  • Internet Explorer only supports EOT
  • Mozilla browsers support OTF and TTF
  • Safari and Opera support OTF, TTF and SVG
  • Chrome supports TTF and SVG.

So what do you do when you only have a single font file? The easiest way is to stop by the Font Squirrel. It offers tons of free @font-face kits that are essentially a one-click download for everything you need to get going with custom, cross-browser compatible fonts. Each kit includes all the font types you need as well as demo HTML and CSS files to get you started.

If FontSquirrel doesn’t have the kit you need, you can also upload your fonts to @font-face generator. All you do is upload your font file and this amazing tool will automatically convert it to all the necessary file types and generate a kit for you to download. The generator provides you with all the font formats that are needed to display correctly in all major browsers. It also provides the css syntax that you can simply copy in the css file.

Note that using the @font-face Kit Generator means that you’re guaranteeing that you have the rights/license to convert and embed the font files you upload to the site. The service is completely free and you can use it with zero sign up.

For example:

Book Your Travel Template theme uses OpenSans font family. The css code looks like this:

				@font-face {font-family: 'OpenSansRegular';
				src: url('../fonts/OpenSans-Regular-webfont.eot');
				src: url('../fonts/OpenSans-Regular-webfont.eot?#iefix') format('embedded-opentype'),
				url('../fonts/OpenSans-Regular-webfont.woff') format('woff'),
				url('../fonts/OpenSans-Regular-webfont.ttf') format('truetype'),
				url('../fonts/OpenSans-Regular-webfont.svg#OpenSansRegular') format('svg');
				font-weight: normal;
				font-style: normal;}

When you have all four font formats, simply copy them into fonts foder and change the above code into this:

@font-face {
font-family: 'YourFontName';
src: url('../fonts/YourFontName-webfont-webfont.eot');
src: url('../fonts/YourFontName-webfont-webfont.eot?#iefix') format('embedded-opentype'),
url('../fonts/YourFontName-webfont-webfont.woff') format('woff'),
url('../fonts/YourFontName-webfont-webfont.ttf') format('truetype'),
url('../fonts/YourFontName-webfont-webfont.svg#YourFontName') format('svg');
font-weight: normal;
font-style: normal;
}

Then just use it like any other font in any other style rule:

p {font-family: 'YourFontName', Helvetica, Arial, sans-serif;}
